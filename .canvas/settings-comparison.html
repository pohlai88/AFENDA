AFENDA Structural Advantages

  • Multi-tenant by design: org-scoped RLS on every settings table, atomic PATCH, effective-value merging of defaults + overrides
  • Custom fields architecture: immutable api_key, value lifecycle decoupled from definition, prospective required enforcement, entity-centric index
  • Feature flag discipline: explicit entitlement vs. behavior categorisation prevents engineering rollout flags polluting org settings
  • Compile-time safety: registry coverage assertion catches missing keys at build time; CI gates enforce contract↔DB parity and schema invariants
  • Audit trail: full per-action audit_log with correlationId, changedKeys array, custom-field lifecycle events — unmatched at this stage

Highest-Value Gaps to Close

  • General depth (Phase 2): company name, timezone, fiscal year, base currency — prerequisite for invoicing and reporting correctness
  • Number series admin (Phase 2): prefix + seed config per document type over existing kernel/execution/numbering infra — no new infra needed
  • Custom fields — Invoice & PO (Phase 4): Supplier is done; extending to transaction entities closes the Zoho/QBO custom-field gap entirely
  • Email settings (Phase 3): SMTP server, from-address, alias routing — required before any outbound workflow goes live
  • Permissions admin surface (Phase 4): RBAC seed data exists; runtime management UI and API are the missing pieces
