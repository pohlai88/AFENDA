Token utilization

Where each token family is used

1. Surface tokens

  • Use surface tokens for spatial depth only.
  • L0 → L4 for importance and containment.
  • Most ERP screens between L1 and L2.

2. Semantic lanes

  • --primary truth, product identity, major CTA.
  • --interactive operator action, navigation.
  • --premium governance, high-trust moments.

3. Status tokens

  • Status only for real workflow state.
  • Soft backgrounds, readable in dense grids.

Usage guideline

Chart tokens for comparison, series, thresholds. Do not reuse status colors as chart palette.

Reserve red/green only where positive/negative is meaningful (variances, gains/losses).

Implementation

How to apply tokens

Recommended mapping

App shell: --background, --sidebar, --topbar

Sections: --surface-100–200

Cards: --card, --surface-275–350

Floating: --popover, --sheet, --modal, --scrim

Spacing: --space-* for premium rhythm.

Rules of discipline

1. Surface hierarchy solves depth before glow or border tricks.

2. Spacing gives luxury; keep --space-section and --space-card generous.

3. Reserve premium for governance and trust, not generic emphasis.

4. Most screens: 80% neutral, 15% semantic, 5% dramatic.
